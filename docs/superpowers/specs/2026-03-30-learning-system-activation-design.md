# 学习系统激活与渐进独立设计

## 概述

激活现有学习管线，使脚本在后台持续学习盒子(HsBox)的打法，逐步进化到可脱离盒子独立运行。采用分阶段渐进策略：先智能降级，再局面评估学习，再特征泛化，最终独立。

## 目标

- 盒子在线时静默学习全维度决策（动作/留牌/选择/局面评估）
- 盒子掉线时用已学策略智能降级
- 一致率稳定 ≥90% 后支持脱离盒子独立运行
- 通用打法，不局限于特定卡组

## 阶段划分

| 阶段 | 目标 | 前置条件 |
|------|------|---------|
| P1 | 接通学习管线，智能降级 | 无 |
| P2 | 局面评估学习 | P1 完成，积累 ≥50 局数据 |
| P3 | 特征评分层，泛化能力 | P2 完成，规则层一致率趋于稳定 |
| P4 | 独立运行门控 | P3 完成，一致率 ≥90% |

## P1：接通学习管线

### 接入点

在 `BotService.MainLoop` 的三个决策点插入采样调用：

| 决策点 | 触发时机 | 采集内容 |
|--------|---------|---------|
| 动作 | 盒子返回 `ActionRecommendationResult` 后 | seed、盒子动作列表、本地AI动作列表 |
| 留牌 | 盒子返回留牌推荐后 | 手牌、敌方职业、先后手、盒子保留/替换、本地保留/替换 |
| 选择 | 盒子返回 discover/choice 推荐后 | 候选卡列表、盒子选择、本地选择、来源类型 |

### 本地AI同步推荐

盒子在线时，**异步**跑一次本地AI推荐用于对比训练，不阻塞主流程。在后台线程完成，耗时不敏感。

### 降级切换逻辑

```
获取盒子推荐:
  ├─ 成功 → 用盒子推荐执行, 同时入队学习样本
  └─ 失败(超时/掉线)
      ├─ 有已学规则匹配 → LearnedStrategyRuntime 生成推荐
      │   └─ 日志标记 [Learned]
      └─ 无匹配规则 → 本地AI (SearchEngine) 兜底
          └─ 日志标记 [LocalAI]
```

### 一致率计算

- 动作一致：盒子的第一个动作（类型+来源+目标）与本地AI相同即算一致
- 滑动窗口：最近 200 个决策点
- 分维度统计：动作/留牌/选择各自独立一致率 + 综合一致率
- 持久化到 SQLite，跨会话连续统计

### 一致率日志格式

每回合输出一行到主日志：

```
[Learning] T8 动作一致 ✓ (盒子:PLAY|EX1_012|0|2 本地:PLAY|EX1_012|0|2) 滑动一致率:87.5%
[Learning] T8 动作不一致 ✗ (盒子:ATTACK|35|42 本地:ATTACK|35|38) 滑动一致率:86.9%
```

## P2：局面评估学习

### 核心思路

从盒子的动作选择中反推评估倾向，调整 `BoardEvaluator` 权重。

### 动作模式与评估信号

| 盒子动作模式 | 推断的评估倾向 | 影响 BoardEvaluator 的维度 |
|-------------|--------------|--------------------------|
| 随从打脸而非解场 | 偏进攻/打脸 | FaceBias ↑、ThreatBias ↓ |
| 随从互换解场 | 偏控场 | BoardControl 权重 ↑ |
| 低费随从优先上场 | 重节奏 | TempoPenalty ↑ |
| 留大牌不出 | 重资源/后期 | HandValue 权重 ↑ |
| 英雄技能而非出牌 | 节约手牌 | AbilityBonus ↑ |

### 评估信号提取

每回合盒子执行完动作后，从动作序列中提取：

- 打脸伤害占比: `face_dmg / total_dmg`
- 交换效率: `消灭敌方随从数 / 损失己方随从数`
- 法力利用率: `mana_spent / mana_available`
- 出牌数 vs 手牌数
- 是否使用英雄技能

### 权重调整方式

指数移动平均（EMA）渐进调整：

```
new_weight = α × observed_signal + (1 - α) × current_weight
α = 0.02
```

按局面分桶存储不同权重组：
- 回合阶段：早期（1-4）/ 中期（5-7）/ 后期（8+）
- 血量态势：优势（领先>10）/ 均势 / 劣势
- 场面态势：领先 / 均衡 / 落后

### 应用时机

`BoardEvaluator.Evaluate()` 调用时，查当前局面所属桶，用该桶的学到权重覆盖 Profile 默认权重。

## P3：特征评分层

### 问题

P1 的规则系统基于"卡牌ID + 局面桶"匹配，遇到未见过的卡牌组合时无规则可用。P3 叠加特征向量评分覆盖此盲区。

### 特征向量（~40维）

**局面特征（~20维）**：

| 特征 | 说明 |
|------|------|
| mana_ratio | mana_available / max_mana |
| friend_minion_count | 己方随从数 |
| enemy_minion_count | 敌方随从数 |
| board_atk_ratio | 己方场攻 / 敌方场攻 |
| board_hp_ratio | 己方场血 / 敌方场血 |
| hero_hp_diff | 英雄血量差 |
| hand_count | 手牌数 |
| has_taunt_enemy | 敌方有嘲讽 |
| enemy_secret_count | 敌方奥秘数 |
| turn_bucket | 回合阶段 |
| pressure_level | 压力等级 |
| lethal_threat | 面临斩杀威胁 |

**动作特征（~20维）**：

| 特征 | 说明 |
|------|------|
| action_type | 打牌/攻击/技能/结束 (one-hot) |
| cost_ratio | card_cost / mana_available |
| is_targeting_face | 是否打脸 |
| is_trading | 是否随从交换 |
| target_has_taunt | 目标是否嘲讽 |
| can_kill_target | target_hp ≤ source_atk |
| remaining_mana_after | 出完后剩余法力 |
| cards_in_hand_after | 出完后手牌数 |
| board_count_after | 出完后场面随从数 |

### 评分模型

线性模型，纯 C# 实现，无外部依赖：

```
score(board, action) = Σ(feature_i × weight_i) + bias
```

- 每个合法动作算一次分，取最高分
- 在线梯度下降更新：pairwise ranking loss `max(0, margin - score_teacher + score_other)`
- 学习率 0.001，每回合更新一次

### 与规则层的融合

```
决策时:
  ├─ 规则层查询 → 有匹配规则?
  │   ├─ 有，且置信度高(权重>1.5) → 直接用规则
  │   ├─ 有，但置信度低 → 规则分 × 0.6 + 特征分 × 0.4
  │   └─ 无匹配 → 纯特征评分
  └─ 输出最终动作推荐
```

## P4：独立运行门控

### 条件（全部满足）

| 条件 | 阈值 | 窗口 |
|------|------|------|
| 动作一致率 | ≥ 90% | 最近 200 个动作决策 |
| 留牌一致率 | ≥ 90% | 最近 50 次留牌 |
| 选择一致率 | ≥ 90% | 最近 50 次选择 |
| 最低对局数 | ≥ 100 局 | 累计总量 |
| 胜率不低于盒子期间 | 差距 < 5% | 最近 30 局 |

### 切换方式

手动切换，不自动。达标后 UI 显示：

```
[学习状态] 独立运行就绪 ✓
  动作: 92.5% | 留牌: 91.0% | 选择: 93.2%
  累计: 347局 | 胜率差: -2.1%
  → 可在设置中开启"独立模式"
```

### 安全机制

独立运行后持续监控胜率。连续 10 局胜率骤降 >15% 时弹出警告，建议重新开盒子学习。

### 持续进化

独立运行后随时可重新开盒子"进修"：
- 开盒子时自动恢复学习模式
- 新数据继续更新规则和特征权重
- 一致率重新计算

---

## 技术约束

- 纯 C# 实现，不引入 ML 框架
- 学习过程不阻塞主循环（异步队列 + 后台线程）
- SQLite 存储，跨会话持久化
- 特征评分模型权重序列化到 SQLite
- 局面评估权重按桶存储到 SQLite
