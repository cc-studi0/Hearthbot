# 战旗单动作对象级就绪门禁 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让战旗招募阶段按“下一条动作对应对象是否已真正可交互”来决定何时执行，而不是继续依赖固定动画等待或泛化全局 ready。

**Architecture:** 先把对象级 ready 的协议与纯规则抽到可测试的共享层，再让 `HearthstonePayload` 负责采集运行时对象状态并调用规则评估，最后由 `BotService` 在战旗招募循环中改成“重定向命令 -> 等该动作对象 ready -> 立即执行 -> 动作后探测”的顺序。保留现有 `BgExecutionGate` 的 `cardId` 真源重定向与动作后状态确认，只替换动作前是否可执行的判断方式。

**Tech Stack:** C# / .NET 8 tests / .NET Framework 4.7.2 payload / xUnit / BepInEx / Unity 反射 / 现有 `PipeServer` 战旗命令链路

---
